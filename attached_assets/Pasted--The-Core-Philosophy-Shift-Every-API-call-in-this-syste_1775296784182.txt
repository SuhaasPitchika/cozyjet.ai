
The Core Philosophy Shift
Every API call in this system must be treated as a conversation with an expert, not a command to a tool. The reason current implementations return generic bloated responses is that the prompts are written like instructions to a machine — "generate three variations of this content for LinkedIn." That framing produces machine output. The prompts in this system must be written like briefings to a senior professional who has deep context about this specific user, their history, their voice, their audience, and their goals. Short sharp context-rich prompts with a clear thinking framework produce dramatically better output than long instruction lists. Every agent prompt in this system follows the rule: less instruction, more context, clear thinking framework, constrained output format.

The second philosophy shift is that the agents must have persistent memory and goals, not just reactive request-response cycles. Each agent wakes up every morning knowing what the user is trying to achieve, what worked last week, what is trending in their niche right now, and what the next milestone in their growth strategy is. Every action any agent takes is evaluated against those goals before execution.

API Stack — ElevenLabs, OpenRouter, Gemini
The three APIs powering this system are chosen deliberately. OpenRouter provides access to multiple frontier models through a single endpoint with intelligent routing, meaning the system can use the best available model for each task without being locked to one provider. Gemini handles deep analytical and strategic tasks because its long context window processes months of user data in a single pass without chunking. ElevenLabs handles voice synthesis for audio content creation and the voice-based Skippy interaction layer, giving the product a genuinely differentiated feature no competitor has — your marketing studio can speak to you and generate audio versions of your content.

Install the following additional dependencies alongside the existing stack: openai configured with base_url pointing to https://openrouter.ai/api/v1 and OPENROUTER_API_KEY as the key (OpenRouter uses the OpenAI SDK format), google-generativeai for Gemini direct integration, and elevenlabs from the official ElevenLabs Python SDK. Add to .env: OPENROUTER_API_KEY, OPENROUTER_DEFAULT_MODEL as anthropic/claude-3.5-sonnet with fallback to google/gemini-pro, GEMINI_API_KEY, and ELEVENLABS_API_KEY with ELEVENLABS_VOICE_ID set to the user's cloned or selected voice ID.

The model routing logic in app/services/model_router.py must initialize an OpenAI client pointed at OpenRouter for all Meta and Skippy calls, a direct Gemini client for all Snooks strategic analysis calls because Gemini's context window handles the data volume better, and the ElevenLabs client for all audio generation. The OpenRouter client must include HTTP-Referer and X-Title headers in every request for proper attribution. The routing service exposes three functions: call_meta_model(messages, max_tokens) using OpenRouter with the configured default model, call_snooks_model(prompt, context_data) using Gemini with the full context loaded, and call_elevenlabs(text, voice_id) for audio synthesis.

Prompt Engineering — Killing Generic Responses
The reason agents return generic bloated content is almost always one of three prompt mistakes: the system prompt is too long and dilutes focus, the user message has no specific context grounding the request, or there is no explicit output constraint forcing concision. Every prompt in this system must follow this exact structure.

The system prompt must be under 200 words, written in first person as the agent's identity, and must include three things only: who the agent is, what it cares about, and how it thinks. Nothing else. No long instruction lists. No "you must always" chains. No formatting rules embedded in the system prompt — those go in the user message as a tight JSON schema the model must match.

The user message must always include the user's voice profile summary in two sentences maximum, the specific task in one sentence, the relevant context as structured data not prose, and the output schema as a JSON template the model fills in. Giving the model a template to fill rather than asking it to create structure from scratch eliminates 80 percent of generic padding.

The temperature setting matters enormously and is almost always ignored. Meta content generation must use temperature 0.85 for creative variation. Skippy summarization must use temperature 0.3 for factual accuracy. Snooks strategic analysis must use temperature 0.5 for balanced analytical creativity. Setting these correctly at the API call level rather than using defaults dramatically improves output quality.

The max_tokens parameter must be set aggressively. A LinkedIn post is never more than 400 tokens. A Twitter thread is never more than 300 tokens. A strategy suggestion is never more than 150 tokens per item. Forcing the model to work within tight token budgets eliminates filler and produces sharper output. If the model cannot express the idea within the budget the idea is not clear enough yet.

Skippy — Autonomous Intelligence Agent
Skippy's system prompt must be exactly this, no longer: "You are Skippy, a work intelligence agent. You read raw activity data from developer and creator tools and find the professional story worth telling. You think like a journalist who covers technology — you look for what is genuinely interesting, novel, or useful to an audience of builders and creators. You write with economy. Every word earns its place. You never pad, never generalize, never produce boilerplate."

The user message template for seed generation must be structured as a JSON object passed as a string: the activity field contains the raw platform data as a compact object, the user_context field contains two sentences summarizing the user's professional identity and current focus area pulled from their voice profile, the recent_seeds field contains the titles of the last five seeds to avoid repetition, and the output field contains the empty template: title as empty string under fifty characters, hook as empty string as one sentence that would make someone stop scrolling, story as empty string as two sentences maximum explaining the work and why it matters, tags as empty array, and platform_fit as empty array. The model fills the template. Nothing else. No prose preamble, no explanation of choices, just the filled JSON.

Skippy's autonomous calendar addition works as follows. When the user says anything in the chat interface that implies a future commitment — "I'm planning to launch this Friday", "I want to start posting about my redesign next week", "remind me to share the results after the experiment" — Skippy's conversation handler must detect the intent using a lightweight classification call to OpenRouter with a small model like mistralai/mistral-7b-instruct at temperature 0.1, extract the implied date, content topic, and platform, and automatically create a calendar_entries record with status draft and a content suggestion pre-populated. It then sends a WebSocket notification: "Added to your calendar: [topic] on [date] for [platform]. I'll remind Meta to draft this the day before." This happens without the user asking for it explicitly. They mentioned it in conversation and Skippy handled it. This is what makes it feel genuinely agentic.

Skippy also runs a daily morning task at 7 AM in the user's timezone that reviews the next seven days of their calendar, identifies any days with no scheduled content, checks the Snooks trend data for relevant topics, and proactively creates draft calendar entries for those gaps with a notification: "Your Wednesday looks empty. I found three relevant things from your recent work and one trending topic in your niche. Drafts are ready in your calendar."

The voice interaction layer uses ElevenLabs. When Skippy responds in the chat interface the response text is simultaneously sent to call_elevenlabs(text, voice_id) and the audio URL is returned alongside the text response so the frontend can auto-play the spoken response. This makes Skippy feel like a real assistant you can have a conversation with rather than a chatbot you type commands into. Users can disable this in settings but it must be on by default for the first session because the first time someone hears their AI agent speak to them in a clear calm voice it is a genuinely memorable product moment.

Snooks — Strategic Growth Agent with Real Memory
Snooks must never give generic advice like "post consistently" or "use relevant hashtags." Every single suggestion Snooks produces must be grounded in the user's actual data. The rule is: if the suggestion would apply equally well to any random person it is not good enough and must be regenerated with a lower temperature and a constraint added to the prompt forbidding generic advice.

Snooks' system prompt: "You are Snooks, a personal brand strategist. You think in growth systems, not one-off tactics. You study patterns in what actually works for this specific person with this specific audience, identify the underlying reasons why, and build repeatable strategies from those reasons. You are direct, data-driven, and contrarian when the data supports it. You never give advice that would apply to anyone — everything you say is specific to this person."

The Snooks strategic analysis call uses Gemini directly because the full context load includes: the last ninety days of post performance data from post_analytics, the last thirty content seeds with their engagement outcomes, the user's growth metrics week over week, the current top twenty trends in the user's detected niches, the user's voice profile, and the user's stated growth goals. This is a large context load that Gemini handles natively in a single call without chunking. The output schema forces Snooks to return a weekly plan as a JSON array of seven day objects each containing a content_opportunity object with topic, angle, platform, optimal_time, predicted_engagement_tier as low, medium, or high based on historical data, reasoning as one sentence referencing specific past performance, and urgency as evergreen, trending, or time-sensitive.

The experiment framework runs as a Celery task snooks_run_growth_experiment triggered weekly. It reviews the last two weeks of content performance, identifies the highest and lowest performing posts, forms a hypothesis about why the top performers worked using a Gemini analysis call, generates two content variations testing that hypothesis for the coming week, schedules them, and adds a snooks_experiments record tracking the hypothesis and expected outcome. Two weeks later it evaluates the result, updates the voice profile with the learning, and presents the finding to the user as a one-paragraph insight: "Your posts that open with a specific number perform 340 percent better than those that open with a question. I've updated your voice profile to prioritize numerical hooks."

Meta — Production Content System with Tuning Integration
Meta's system prompt must be the most carefully crafted of the three because it is the one generating public-facing content. The system prompt must be: "You are Meta, an elite content writer who specializes in making technical and creative work accessible and compelling to broad audiences. You write in this person's exact voice — their rhythm, their vocabulary, their level of formality, their humor. You never sound like AI. You never use hollow phrases like 'dive into', 'game-changer', 'in today's fast-paced world', or 'it's important to note'. You write the way a smart, specific, self-aware person writes when they are at their best. Every piece of content you write has a reason to exist — a specific insight, a specific story, or a specific provocation that earns the reader's attention."

The tuning system is what makes Meta genuinely personalized over time and it must be implemented with real depth. Every file added to the Tune page — every writing sample, every approved post, every edited variation — must be processed by a background Celery task meta_process_tuning_file that does the following. It reads the content, calls OpenRouter with the model anthropic/claude-3-haiku at temperature 0.2 with a prompt asking it to extract five to ten specific stylistic observations about this writing as a JSON array — observations like "uses sentence fragments for emphasis", "opens with a specific scene rather than a general statement", "references concrete numbers rather than vague quantities", "uses humor through understatement", "asks rhetorical questions mid-paragraph." These observations are saved to a tuning_observations table keyed to the user. When Meta generates content the top fifteen tuning observations ranked by frequency across all tuning files are injected into the user message as the style_rules field in the output template. The model sees these not as instructions but as descriptions of how this person writes, and it mirrors those patterns.

The viral hook generation works as a separate pre-generation step. Before Meta writes the full content it runs a hook generation call with temperature 0.9 — high creativity — asking specifically for the opening line only. The prompt for this call includes the seed topic, the platform, three examples of the user's highest performing opening lines from post_analytics, and the constraint that the hook must create either curiosity, tension, surprise, or a strong opinion in under fifteen words. Five hook options are generated. Meta then uses the best-scoring hook as the opening of the full content generation call. Separating hook generation from body generation with different temperature settings produces dramatically better results than trying to get both from a single call.

The content generation for each platform must use separate API calls rather than asking for all platforms in one call. One call per platform with platform-specific temperature: LinkedIn at 0.75 for professional creativity, Twitter at 0.90 for punchy distinctiveness, Reddit at 0.65 for detailed accuracy, Instagram at 0.85 for visual and emotional resonance, YouTube at 0.70 for structured clarity. Parallel calls using Python asyncio.gather() so all five platforms generate simultaneously and the total latency is the time of the single slowest call rather than the sum of all five.

ElevenLabs Integration — The Differentiating Feature
ElevenLabs serves two purposes in CozyJet. First it powers Skippy's voice responses in the chat interface making the agent feel like a real conversational presence. Second and more importantly it generates audio versions of all content which is a feature no competitor offers — a LinkedIn post can become a voice memo the user sends to their network, a Twitter thread can become a podcast segment, a YouTube script becomes a complete voiceover.

The POST /api/elevenlabs/generate-audio endpoint accepts a content_id and an optional voice_id defaulting to the user's configured voice. It loads the content text, cleans it for speech (removes hashtags, converts at-mentions to natural language, expands abbreviations), calls the ElevenLabs API with the eleven_turbo_v2 model for speed, streams the audio back as a binary response, and saves the audio URL to the content record. The voice cloning feature on the Ultra plan allows users to upload three to five voice samples via POST /api/elevenlabs/clone-voice, creates a cloned voice via the ElevenLabs API, saves the voice_id to the user profile, and all subsequent Skippy audio responses and content audio use their own voice. Hearing your AI agent speak in your own voice and hearing your social media posts read back in your own voice is the single most memorable moment in the product.

The Autonomous Loop — What Makes It Revolutionary
The system that ties everything together and makes CozyJet genuinely different from every other content tool is the autonomous growth loop that runs without any user interaction. At 6 AM every day in the user's timezone a Celery chain executes in sequence. Skippy syncs all integrations and generates seeds from overnight activity. Snooks evaluates the week's content calendar, checks for gaps, checks the fresh trend data, and fills any empty days with draft content entries. Meta runs in background generation mode, taking all draft calendar entries without content attached and generating the content for each one using the user's voice profile and the latest tuning observations, saving everything as draft status. At 7 AM Skippy sends the user a daily briefing via WebSocket and ElevenLabs audio: "Good morning. You shipped two things yesterday worth talking about. I've drafted four posts — two for LinkedIn, one thread for Twitter, and one Reddit post. Snooks spotted a trending conversation about authentication security that matches your recent work. Everything is in your calendar ready to review. Your best time to post today is nine AM." The user opens the app, sees everything already prepared, spends three minutes reviewing and approving, and closes the app. Their marketing is handled. That is the revolutionary version of this product.
